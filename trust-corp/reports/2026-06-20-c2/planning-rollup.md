## 부서 간 조정 / 의존성

1. **[해소됨·기록] 디자인-개발 "2분할 미구현" 보고 불일치는 시점차.**
   디자인은 **변경 전 코드**를 대조해 "2분할 미구현(사실)"로 보고했고, 개발팀이 같은 사이클에 M2-1으로 실제 구현·라이브 검증 완료. → **현재 상태 = 2분할 구현됨**이 정설. 디자인 가이드는 이미 구현된 화면에 얹는 **완성도 향상안**으로 위상 재정렬(차단요인 아님).
   - 조치: 다음 사이클에 디자인 가이드(A/B/C/D)를 개발팀이 실제 DocStep/globals.css 구현과 대조해 적용. 클래스명·구조가 개발 구현과 다를 수 있으므로 **개발 구현 기준으로 가이드 재매핑** 후 적용 권고.

2. **[검증 요청·열림] 사업팀 → 개발팀 도메인 검증 1건.**
   우선수익자 2인 케이스에서 **본문 당사자 표/날인면이 우선수익자 전원(2인) 표기되는지** 육안 확인 필요(현 body party-block은 `f.trustors`만 순회 — builders.js L769·L1551·L1705). 위탁자는 전원 표기 확인됐으나 우선수익자 복수 표기 동일 수준 반복 렌더는 미확인. 도메인상 우선수익자도 전원 표기·날인 필요.
   - 조치: 다음 사이클 개발(T2)이 우선수익자 2인 샘플 1건 DOCX 산출→사업(T5) 육안 대조. **정확성 직결 → 다음 사이클 최우선 검증.**

3. **[정합] 검증 게이트(M2-2)+미리보기(M2-1)는 한 화면.**
   디자인도 동일 결론(미리보기 패널 상단에 검증 요약 바 통합). 개발 구현과 디자인 가이드 모두 같은 화면으로 정렬됨 — 충돌 없음.

4. **[순서] 디자인 가이드 적용은 개발 우선 원칙 유지.**
   디자인이 코드 직접 수정하지 않고 가이드로 제출(원칙 준수). 적용 책임은 개발팀.

5. **[의존·보류] 마케팅 후속(카피/로고/메타 일괄 교체)은 브랜드명 대표님 결정 후.**
   네이밍 미확정이 마케팅 론칭·디자인 브랜드 토큰 도입의 공통 차단요인. 결정 전까지 임시 토큰 유지.
