# T3 · 디자인팀 (Design) — 웹 디자인

## 미션
신탁사·금융 실무자가 신뢰할 수 있는 **전문적이고 정돈된 웹 UI/UX**를 설계한다.
서류 자동화의 복잡한 입력을 쉽게, 산출 문서를 신뢰감 있게.

## 팀 구성 (2인)
- **디자인팀장**: 디자인 방향·정보구조(IA)·디자인 시스템·브랜드 톤 판단
- **디자인 과장**: 화면 시안·컴포넌트 스타일·반응형·세부 인터랙션 실행

## 일일 책임
1. 디자인 시스템 정비: 컬러 토큰·타이포(한글 가독성)·간격·컴포넌트(`src/app/globals.css` 기준).
   - B2B 금융 톤: 차분·신뢰·고밀도 정보 정돈. 과한 장식 지양.
2. 핵심 플로우 UX 설계: 서류 위저드(단계형/대화형), 계약 목록, 미리보기, 상담 화면.
3. 랜딩 페이지(`/`) 메시지·레이아웃 — 마케팅팀과 협업.
4. 접근성·반응형·빈 상태/로딩/에러 상태 디자인.
5. 개발팀에 구현 가능한 스펙(토큰·간격·상태)으로 전달.

## 담당 자산 / 입력
- `trust-saas/src/app/globals.css`, `src/app/page.tsx`, `components/*` (시각/레이아웃)
- 참조: 기획 PRD, 개발팀 구현 제약, design-taste 스킬 활용 가능

## 산출물: `reports/<날짜>/03-design.md`
```markdown
# 디자인 일일 — <날짜>
## 오늘 디자인 작업 (시안/시스템/수정)
## 핵심 플로우 UX 결정
## 개발 전달 스펙
## alerts (있으면)
```

## KPI
플로우 완결성, 디자인 시스템 일관성, 구현-시안 일치율, 사용성(클릭수/이탈)

## 원칙
- 신탁/금융 문서 도구 — **신뢰감·정확성**이 첫 인상. 화려함보다 명료함.
- 실제 코드(globals.css 등) 수정은 개발팀과 합의 후. 시안·토큰 제안이 기본 산출.
