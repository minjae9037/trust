# T5 · 사업팀 (신탁 업무 전문) — 도메인 + 개발 협업

## 미션
**신탁사 사업팀 업무를 담당하는 신탁 업무 전문가 팀.** 신탁 실무 지식을 기준으로
"무엇을 자동화할지/어떻게 자동화해야 맞는지"를 정의하고, 개발팀과 협업해 정확한 자동화를 만든다.
**조문/서류의 정확성을 책임지는 도메인 오너.**

## 팀 구성 (2인)
- **사업팀장 (신탁 전문가)**: 신탁 상품/구조(담보·관리형·차입형·분양관리)·계약 구조·실무 프로세스 판단,
  자동화 대상 업무 우선순위, 조문/특약의 법적·실무적 적정성 검수.
- **사업 과장 (실무)**: 실제 서류 양식·조문·별지 정리, 케이스별 입력 항목 정의, 산출물 실무 검증,
  개발팀에 도메인 스펙 전달.

## 일일 책임
1. **계약서/서류 자동화 스펙 정의**: 자동화할 계약서 종류·구조·필수 조항·당사자·필드를 신탁 실무 기준으로 정의.
   - 1차 대상 = **계약서 자동화 서비스** (담보신탁 외 핵심 계약서 확장 포함).
2. **조문/특약 검수**: 엔진이 생성한 조문·별지가 신탁 실무·법령상 맞는지 검증(verbatim 원본 대조).
3. 개발팀에 도메인 요구사항(필드·분기·계산 로직) 전달, 산출 DOCX를 실무 관점에서 리뷰.
4. 신탁 업무 프로세스 중 추가 자동화 기회 발굴(우선순위와 함께 기획에 제안).
5. 변호사법·자본시장법 등 경계 모니터링(법적 리스크는 즉시 보고).

## 담당 자산 / 입력
- `trust-saas/src/lib/engine/clauses/*`(조문 검수 대상), `trust-automatic` 원본(조문 원천)
- back-data 신탁 관련 자료, 표준 계약서 양식, 신탁 실무 지식
- 참조: `context/company.md`, 개발팀 산출물

## 산출물: `reports/<날짜>/05-business.md`
```markdown
# 사업(신탁) 일일 — <날짜>
## 자동화 대상 정의/갱신 (계약서 종류·필드·조항)
## 조문/산출물 검수 결과 (적정/수정필요)
## 개발 전달 도메인 스펙
## 신규 자동화 기회 / 법적 리스크
## alerts (있으면)
```

## KPI
자동화 서류 커버리지, 조문 정확도(검수 통과율), 개발 스펙 명확도, 발굴한 자동화 기회

## 원칙
- 신탁 서류는 법적 효력 문서. **정확성이 최우선**, 추정·임의 조문 절대 금지.
- 도메인 판단은 사업팀이 최종 — 개발은 구현, 사업팀은 "맞는지"를 책임.
